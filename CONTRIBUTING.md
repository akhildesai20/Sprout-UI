# Contributing to Sprout UI

Lightweight guide for contributors.

## Run locally

```bash
npm install
npm run build
```

- **Demo:** `npm run dev` then open the URL shown (serves repo root; demo at `/demo/`).
- **Docs:** `npm run docs:dev` then open the local URL.
- **Benchmarks:** After `npm run build`, serve the repo root (e.g. `npx serve .`) and open `/benchmarks/baseline.html`, `sprout-components.html`, `sprout-interactions.html`.

## Where things live

| Path | Purpose |
|------|--------|
| `src/sprout.css` | Core CSS (tokens, components). Single source of truth. |
| `src/sprout.js` | Core JavaScript (theme, modal, toasts, tabs, dropdown, etc.). |
| `scripts/bundle.js` | Builds `dist/sprout.bundle.js` from minified CSS + JS. |
| `scripts/scope-sprout-css.js` | Generates `docs/.vitepress/theme/sprout-scoped.css` from `src/sprout.css`. |
| `dist/` | Built assets (min CSS, min JS, bundle). Tracked so clone-and-run works. |
| `docs/.vitepress/theme/sprout-scoped.css` | **Generated.** Do not edit by hand. Regenerate with `node scripts/scope-sprout-css.js` after changing `src/sprout.css`. |

## Generated files

- **`docs/.vitepress/theme/sprout-scoped.css`** — Generated from `src/sprout.css` by `scripts/scope-sprout-css.js`. Run the script after CSS changes so docs demos use the latest styles.
- **`dist/*`** — Generated by `npm run build`. If you change `src/sprout.css` or `src/sprout.js`, run `npm run build` before testing demo/benchmarks.

Do not edit generated files blindly; update source and regenerate.

## Benchmarks

The `benchmarks/` folder is for measurement only: baseline vs Sprout components vs Sprout interactions. No refactors to Sprout core are done there. Record results in `benchmarks/results.md`. See `benchmarks/README.md` for how to run and record.

## Spec

Design and behaviour are defined in `SPROUT.md`. When in doubt, refer there.
